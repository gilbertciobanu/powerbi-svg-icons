# DAX Measure Generation Workflow

Read `icon-specs.md` first if you haven't already — it contains the canonical rules this workflow depends on (single quotes, hardcoded color, template header).

This workflow converts approved SVG icon files into Power BI DAX measure strings.
Run this **after** icon creation is complete and the user has approved the final SVG versions.

---

## How Power BI Consumes SVG Icons

Power BI cannot reference local SVG files directly. Instead, the SVG markup is embedded
as a data URI string inside a DAX measure. Power BI then renders it as an image when
the measure's Data Category is set to "Image URL".

**Data URI format:**
```
"data:image/svg+xml;utf8," & "<svg ...>...</svg>"
```

**Data Category setting (required):**
In Power BI Desktop: Modeling tab → select the measure → Properties pane → Data category: Image URL.
Without this setting, Power BI displays the raw string instead of rendering the image.

## DAX Measure Structure

```dax
Icon Name =
-- icon-[name].svg | YYYY-MM-DD HH:MM:SS
    "data:image/svg+xml;utf8," &
    "<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 96 96' width='100%' height='100%' fill='#1C3879'>" &
        "<path d='...'/>" &
    "</svg>"
```

The string is split by logical grouping — the data URI prefix, the SVG opening tag, shape
elements, and closing tag each on their own line. This makes the structure readable and individual
segments easy to edit. Line breaks between `&` concatenations are safe; DAX ignores whitespace
between string segments.

**Rules for DAX-safe SVG:**
- All SVG attributes must use **single quotes** — see `icon-specs.md` section 8 for the full rule, including how to handle externally sourced SVGs that use double quotes.
- Strip the SVG comment header — it adds dead weight to the string with no runtime value. Instead, add a DAX comment on the line immediately after `=` with the source filename and the `Created` timestamp read from the SVG comment header (see Output File Format below).
- Split the string by logical grouping as shown above — one segment per line, indented for readability.
- Encode `&` as `&amp;` if it appears anywhere in the SVG content (rare, but occasionally appears in `<title>` or metadata elements).
- The `xmlns` attribute is required for the data URI to render correctly in all Power BI versions.

**Validation step — run before generating DAX for any externally sourced SVG:**
Before converting an SVG that was not generated by this skill (e.g. downloaded, shared by another user, or sourced from a library), check for the following and warn the user if found:
- `<script>` tags — flag as potentially unsafe, do not proceed without user confirmation
- `href`, `src`, or `xlink:href` attributes pointing to external URLs other than `www.w3.org` — flag each one and ask the user whether to strip or abort
- `<image>` tags with external `href` — same as above
- `<use>` elements referencing external symbol libraries — these will not resolve in Power BI's sandbox; flag and suggest inlining the referenced shape

Skip this check for icons generated by Workflow A — they are already clean.

## Generation Process

### Single icon
1. Read the `.svg` file from the `icons/` subfolder. Extract the `Created` field from the SVG comment header — this becomes the version stamp in the DAX comment, identifying which version of the SVG the DAX was generated from. If no `Created` field is found, resolve the timestamp using the same priority order as icon-specs.md section 6: filesystem `mtime` first, current datetime as a last resort. Note to the user which source was used.
2. Convert the SVG to a DAX measure string using the structure above; verify it complies with all rules in `icon-specs.md` before proceeding
3. Use the SVG filename (without extension) as the measure name, formatted in Title Case
   - `icon-filter-regular.svg` → measure name `Icon Filter Regular`
   - `icon-filter-filled.svg` → measure name `Icon Filter Filled`
4. Save the output to `icons/icon-[name]-[variant]_dax.txt`, overwriting any existing file. If a file was overwritten, note it explicitly in the output (e.g. "Updated existing `icon-filter-regular_dax.txt`")
5. Show the full DAX string in the conversation for the user to copy

### Batch processing
If the user asks to process multiple icons (e.g., "convert all icons to DAX" or provides a list):
1. List the `.svg` files that will be processed and confirm with the user before starting
2. Process each file in sequence using the single-icon steps above
3. After all files are done, summarize: list each measure name and its corresponding `.txt` file

**When confirming a batch**, show the list like:
```
Ready to generate DAX measures for:
  • icon-filter-regular.svg   → icon-filter-regular_dax.txt   (measure: Icon Filter Regular)
  • icon-reset-regular.svg    → icon-reset-regular_dax.txt    (measure: Icon Reset Regular)
  • icon-export-regular.svg   → icon-export-regular_dax.txt   (measure: Icon Export Regular)
Proceed?
```

## Output File Format

Each `_dax.txt` file contains exactly one measure definition, ready to paste into Power BI's
DAX editor or a calculation group. The DAX comment identifying the source SVG file goes on the
line immediately after `=`. Example file content:

```
Icon Filter Regular =
-- icon-filter-regular.svg | 2026-02-27 14:30:00
    "data:image/svg+xml;utf8," &
    "<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 96 96' width='100%' height='100%' fill='#1C3879'>" &
        "<path d='M3 6h18v2L13 14v5l-2-1v-4L3 8V6z'/>" &
    "</svg>"
```

No extra text, no markdown — just the measure name, DAX comment, and the grouped string value.
Easy to copy and paste directly into Power BI Desktop without cleanup.
